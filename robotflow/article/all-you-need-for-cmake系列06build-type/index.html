All You Need for CMake系列06——Build Type

准备代码

代码结构如下：

.
├── CMakeLists.txt
├── main.cpp

main.cpp:

#include <iostream>

int main(int argc, char *argv[])
{
   std::cout << "Hello Build Type!" << std::endl;
   return 0;
}

CMakeLists.txt下面的内容如下：

cmake_minimum_required(VERSION 3.5)

# Set a default build type if none was specified
if(NOT CMAKE_BUILD_TYPE AND NOT CMAKE_CONFIGURATION_TYPES)
  message("Setting build type to 'RelWithDebInfo' as none was specified.")
  set(CMAKE_BUILD_TYPE RelWithDebInfo CACHE STRING "Choose the type of build." FORCE)
  # Set the possible values of build type for cmake-gui
  set_property(CACHE CMAKE_BUILD_TYPE PROPERTY STRINGS "Debug" "Release"
    "MinSizeRel" "RelWithDebInfo")
endif()

# Set the project name
project (build_type)

# Add an executable
add_executable(cmake_examples_build_type main.cpp)

编译流程

mkdir build
cd build
cmake ..
make

设置Build Type同样有两种方式，一种是cmake .. -DCMAKE_BUILD_TYPE=Release 一种就是如上述CMakeLists文件一样加上那么一段。

可用的build type有如下几种：

  • Release
  • Debug
  • MinSizeRel
  • RelWithDebInfo 具体含义还是比较容易看明白的。
