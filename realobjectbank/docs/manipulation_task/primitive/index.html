    Primitives #

    本文给出的Primitives，省去了PreCondition和EndCondition的描述，在PrimitiveNet里给出了部分sample，详细的请找Wenqiang要数据库。

    因此，我们在这里主要关注Contact和constraint。

    0-Point Primitives #

    为了完备，当然先说一下没有Contact的情况，那就是0-Point的Primitive。那么这个定义有没有意义呢？当然是有的，无论是视觉伺服，还是无处不在的Monitor，对于task而言和对象都是无物理接触的。

    1. eye contact
    2. ear contact
    3. status check
    4. move

    1-Point Primitives #

    1-Point Primitive表示这个primitive最少可以用1个contact点完成，但更多的点当然也是ok的。当然鉴于实际的机械爪都胖胖的，所以实践中肯定也不只一个点了。

    1. Push：单点情况下的push，通常也叫poke
      • trajectory constraint：包括从A点把sth/some point推到B点；假装推A但没推动等。
      • force constraint：Spread
    2. Pull：
      • trajectory constraint
    3. Flip/Roll
    4. Release：考虑一下指尖转篮球，然后把它放掉，这就是单点的release

    2-Point Primitives #

    同理，更多的点完成这些primitive也是ok的。

    1. Grasp
      • trajectory constraint
      • force constraint

    虽然目前最强的机械手有24个自由度，但在不考虑in-hand manipulation的情况下，我们定义到2-point primitive就行了。更多的点主要是提供了：

    • 冗余度，保持稳定
    • in-hand调整的可行性
    • 某些奇妙抓型的可能性（比如握笔，握筷子）

    需要注意的是，尽管我们多次说了在这个定义中暂时不考虑in-hand manipulation，但日常生活中，in-hand manipulation其实是无处不在的，包括握笔写字，虽然它也可以在不dexterous的情况下完成，但一定是少了笔头摇曳那味儿。我们会在Task章节里收录一些in-hand manipulation，并也尝试尽可能把它们放进这套框架里。

    几个延续问题 #

    • 单手的atom primitive是不是真的只有这么几个？
      • 我觉得是，可能有漏网之鱼，但不会多。
    • 双手是否有primitive？
      • 通常复杂性的上升一定会带来一些群体性的特性，但我想了很久确实难以找出一个有意义的双手专属primitive。
