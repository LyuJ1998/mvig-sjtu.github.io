    Concept #

    Concept是一个很抽象的词，什么都可以描述concept。我这里根据task的定义也整理一套concept出来，当然不代表最终意见，会反复修改。

    Contact-related Concept #

    人类造了很多词用于描述不同的动作，但这些词对应的动作，其力学pattern可能是类似的。因此，我们构建了一个physics-based近义词表，这个表将从源头上把一些人类认知的词整理到一个集合上去，并用一个词作为该集合的代表。比如push和poke的关系。

    此外关于contact的point这边，affordance要依托object knowledge（所以为啥把task zoo的事情也放在object knowledge bank这个网站，这就是原因。反过来object knowledge也要依托于primitive的定义，因为我们对primitive的定义也是基于认知的）。

    需要注意的是，affordance带来的contact通常是constraint-oriented contact，也就是它的语义信号足够强，其中也蕴含了contact完后该怎么运动的信息。

    Constraint-related Concept #

    我们说了constraint主要有三种类型，即trajectory，force和object affordance。 幸运的是，trajectory本质上是几何的，可以用简单的参数来表示；force本质上是力学的，不考虑振荡的情况也是可以描述的。
