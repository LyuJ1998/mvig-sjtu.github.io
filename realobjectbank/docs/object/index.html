    Object #

    物体是Robot Perception的中心，Robot Vision区别于General-purposed Computer Vision一个很大的区别就是Object-centric。一个直接的要求，就是需要能接触到物体。

    目前来说，接触物体有两种方式，一种是通过仿真，一种是通过线下扫描。这两种数据的优缺点我就不多说了。总之我们都提供支持。

    物体的标注格式（updating~） #

    有什么建议可以跟我说。

    标注项 标注格式 备注
    类名/category 文本 可以结合wordnet形成taxonomy
    几何模型/3D model mesh/sparse point cloud 如果mesh拿不到，可以用sparse点集代替
    建模方式/modeling method 文本 cad or scan
    扫描仪型号/scanner model 文本 只适用于建模方式为“scan”的物体
    结构/structure keypoint 具体怎么定义和组织keypoint还是一个open的问题，当前采用 You Yang的方法
    纹理/texture 图片
    材质/material 图片
    物理属性/physics property 数值 包括但不限于：质心，重量，惯量
    物体类型/type 文本 rigid，articulated，flexible，tearable
    关节/joint 数值 只针对articulated物体：joint location, link length, angle limits, dynamics property等
    控制点/control point 坐标 只针对flexible物体：格式暂定。
    可撕裂区域/tearable region 热力图 只针对tearable物体：格式暂定。
